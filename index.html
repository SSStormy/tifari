Tifari

An in-browser, taggable imageboard for artist reference images.

Self-hosted, fully open source.

Setup

Getting tifari

You can find both Windows and Linux downloads on the GitHub releases page here.

Once you have that downloaded, all you need to do is place the files somewhere that's easy to access and run tifari.exe

tifari.exe is in charge of serving the website, files and managing all the tags. Launching it will open a command shell. As long as that is open, tifari is running and can be accessed by visiting "localhost:3555" on a modern browser.

First time setup

On your first launch, you'll have to point tifari to an image folder. This folder should contain images that you want to tag using tifari.

The initial scan of the image folder could take a while. It's safe to close the tab and check back later.

Usage

Layout

Tagging

Hovering over an image will reveal a "Show tag list" button. Clicking on it will show a new menu. This is the tag editor.

The tag editor lists all of the tags that tifari knows. Adding tags to tifari is done by adding a new tag to an image.

You can add tags to an image in the tag editor by marking the checkbox of an existing tag or typing in the name of a tag into the "New tag" text box.

You can remove a tag from an image by unmarking the tag's checkbox.

A tag with no images that use it will be automatically removed from tifari.

Searching

Searching is done by typing in a query into the "Search by tags" text field.

Queries consist of any combination of "must contain" search tags and "must not contain" search tags

"Must contain" tags are simply just tags. If we have a tag called "head" then the "must contain" tag is simply "head". This type of search tag will match images that contain the given search tag.

"Must not contain" tags are tags that are prefixed with a dash ("-"). If we have tag called "head' then the "must not contain" tag is "-head". This type of search tag will match images that DO NOT contain the given search tag.

Advanced

Config

The config.json file tells tifari what addresses/ports to bind the frontend and API to, as well as where to look for images and the database

The schema for the config.json can be found here

API

The frontend communicates with the rust backend via a web API. By default it is hosted at 127.0.01:8001. The default hosting address can be changed in the config.
